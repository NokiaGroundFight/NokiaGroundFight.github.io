## AI & Knowledge

Assume you join "a project", you know nothing about it but start writing code for it.
You do it the old way, writing by hand instead by an LLM.

After a while you run into a problem: \
"How do I check for X?" you ask. \
"I don't want to write the whole thing again. Surely, someone else had to check X before as well. - Let's see what they did!"

So you venture out to other peoples code and read it.

And of course, the others solved the problem. \
They even did it in a non-naive way. \
It is a little more efficient. \
It will be a little easier to maintain.

Finding it was annoying, sure. \
But now you also have a rough idea about 5 other things happening in "a project".

So you think "good good good", use their solution to check X and go on.

\-

With the question you asked something happened. \
You didn't just solve your problem, you learned. \
You learned how checking X can be done a little more efficient, a little more maintainable. \
And you came around the 5 other things which you will forget the moment you part ways with "a project".

"a project" gained something as well. With you exploring, it gained a tiny bit of institutionalized knowledge. \
You know, the kind of knowledge which will solve problems a little more efficient, a little more maintainable.

And yet, here we are. \
The LLM generates the code. \
The LLM generates the check for X. It generates an isolated check. Not efficient, not maintainable. \
The LLM doesn't search if someone did it already. Except some input asks it to. Maybe it does, maybe not - who knows... \
The LLM doesn't learn, it doesn't grow the institutionalized knowledge, it generates based on input. \
Doesn't matter though, because the input has to be small. The knowledge won't fit in. And even if it could, it's rarely written down anyways.

So we generate code, it is so comfortable. \
Of course we read it, to make sure it makes "sense". \
We, who didn't even know how to check for X. \
And so we conclude "it makes sense". The test passes. The feature works. \
The minute we spent to read it - it all seems fine.

At some point we stop reading as well. Because the test passes. Because the feature works.

No one asks how to check X a little more efficient, a little more maintainable. \
No one asks "isn't this check for X a little odd?" \
No one asks any question anymore.

No one grew our knowledge.
